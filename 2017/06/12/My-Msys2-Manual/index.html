我的Msys2应用手册

这编手册主要是记录我在Msys2上安装一些Unix-like软件的过程。这些软件通常是在
Msys2下用pacman装不到的，或者要编译，或者要针对Msys2改代码。一方面可以记录
下来备忘，另一方面可以让有兴趣的同学可以参考安装这些软件。

Msys2说明及安装

Msys2说明

Windows下的Unix-like环境主要有两种，Cygwin和Mingw。前者是嵌入中间层，转换
windows的API为posix的API，程序是unix程序用的是posix的API，当然有这样一层
中间层，效率会低一些，而且兼容性则不错。Mingw则是直接使用windows的API，
效率比前者好，但是可能要改源码以适应windows下的API。

而Msys2则是结合上述两者的优点。用Cygwin的一些Unix工具，又提供mingw的一些
高效程序。不考虑交叉编译，Msys2基本分成Msys2、Mingw32、Mingw64三个部分，
前者安装时不带前缀比如git，而Mingw32和Mingw64分别用mingw-w64-i686-和
mingw-w64-x86_64-做前缀。比如Python2，三个部分的都有一个，就我的32位系统
而言，我是用mingw-w64-i686-python2。这三个部分都有相应的入口：程序的根目录
中的msys2.exe、mingw32.exe和mingw64.exe。作用相似，都是配置好环境变量，特别
是PATH，msys2.exe是不会添加mingw32的程序目录（/mingw32/bin）到PATH中，
mingw32是不会添加mingw64的程序目录（/mingw64/bin）到PATH中。

多线程返回问题

###中文乱码问题

问题的主要原因是中文windows是用gbk作默认编码，而如果gvim上用gbk，youcompleteme又会报错。
而如果gvim里用utf-8，而InstantRst的网页则是乱码。这里给出一个临时方案，
因为只对0.9.2.2(直接用pip install instant_rst装的）,如果用github连接装的0.9.3.2则些方案报错。
修改/mingw32/lib/python2.7/site-packages/instant_rst/server.py

似乎新的版本修复了这个乱码问题。
